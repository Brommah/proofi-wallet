### Capability Token Architecture

```typescript
interface CapabilityToken {
  issuer: string;      // user's wallet address
  grantee: string;     // agent's public key
  scope: string[];     // ["health/*", "prefs/ui"]
  permissions: string[]; // ["read", "write"]
  expiry: number;      // unix timestamp
  wrappedDEK: string;  // DEK encrypted for agent
  signature: string;   // signed by user
}
```

**Flow:**
1. User data → encrypted in browser → DDC (ciphertext only)
2. User grants capability token to agent
3. Token contains wrapped DEK (X25519 encrypted)
4. Agent unwraps DEK → decrypts data → processes
5. User revokes = new DEK, agent's token invalid
