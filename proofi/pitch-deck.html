Security Model
Zero-Knowledge Architecture
The server never sees your private key. Even if every server is compromised, your keys remain safe.
üîê

Key Derivation

Private keys are derived entirely client-side using PBKDF2 (100K iterations). The server provides a deterministic salt, but without the user's PIN, it cannot reconstruct the key.

Server knowsemail, salt
User knowsemail, PIN, salt
Can derive keyUser only ‚úì
‚úçÔ∏è

Signature Auth

Every credential is signed with sr25519 (Schnorrkel). Signatures are verifiable by anyone with the public key. No central authority needed.

Algorithmsr25519 / ed25519
JWT tokensES256
Salt derivationHMAC-SHA256

Decentralized Storage

Credentials stored on Cere DDC (Bucket #1229). Content-addressed via CID. No single point of failure. Data persists even if Proofi servers go offline.
